prometheus-operator add coredns monitor

Published on Monday, January 14th 2019 at 11:38
Last updated on Tuesday, January 15th 2019 at 12:37

参考文档：

https://devops.college/prometheus-operator-how-to-monitor-an-external-service-3cb6ac8d5acb
https://github.com/coreos/prometheus-operator/issues/2110

1.创建coredns 监控的endpoints：

增加内容如下：
k8s-app: kube-dns
component: coredns

  • name: http-metrics-coredns
    port: 9153
    protocol: TCP
    targetPort: 9153
    整个kube-dns service 内容如下：
    kubectl edit svc kube-dns -n kube-system

2.增加serviceMonitor 资源：

prometheus中增加coredns 的ServiceMonitor：
kubectl apply -f kube-prometheus-exporter-coredns.yaml

3.为什么不会自动增加到prometheus 的target中:

查看ServiceMonitor 的匹配规则：

匹配存在k8s-app label 的ServiceMonitor ，会自动加到prometheus 的监控target中

4. 查看prometheus 的target是中是否有coredns：

5. 增加grafana 中coredns 的dashboard模板：

我选用的模板地址：https://grafana.com/dashboards/9494
查看效果：

6. 增加coredns 的报警规则：

Kubectl apply -f prometheus-k8s-rules.yaml -n monitoring
文件路径：/prometheus-operator/contrib/kube-prometheus/manifests/prometheus/prometheus-k8s-rules.yaml

注意:job 的名字coredns：是通过ServiceMonitor 中的joblabel 标签指定的值是component，在service 中的标签component 对应的值为coredns

8. 查看coredns报警规则是否生效：

查看prometheus 的配置文件加载是否成功：
1）查看日志：

2）查看日志重新加载是否有报警
从prometheus 的alerts 页面中，查看是否有”prometheus_config_last_reload_successful “ 报警
打开prometheus-》alerts

9. 完
