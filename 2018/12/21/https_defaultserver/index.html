https default server

Published on Friday, December 21st 2018 at 17:18
Last updated on Friday, December 21st 2018 at 17:53

1.前提条件：

我们的nginx 服务器只配置了一个https协议的子域名：https://api.example.com 一个http协议的域名，http://test.example.com 这两个域名的后台服务是同一个服务，只不过版本不同。api.example.com 返回的是正式数据，test.example.com 返回的是测试数据。

2.现象：

开发同学测试的时候使用的是https://test.example.com/xxxx?xxx. 发现能正常返回数据，但是这个服务应该是由api.example.com 服务返回的正式数据。

3.分析：

难道nginx 不校验server_name 的配置？ 由于ssl 443 协议的只有api.example.com ,当nginx 找不到https 协议的虚拟主机https://test.example.com 就由https://api.example.com 来处理返回。给人一个误导。

4.解决方案：

给nginx 设置一个默认443端口的虚拟主机：
server {
listen 80 default_server;
listen 443 ssl default_server;
server_name _;
ssl_certificate cert/api.example.com.pem;
ssl_certificate_key cert/api.example.com.key;
ssl_session_timeout 5m;
ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:ECDHE:ECDH:AES:HIGH:!NULL:!aNULL:!MD5:!ADH:!RC4;
ssl_protocols TLSv1 TLSv1.1 TLSv1.2;
ssl_prefer_server_ciphers on;
return 403;# return 444;
}

注意：

必须配置下面关于ssl证书的配置。否则nginx会报：
no “ssl_certificate” is defined in server listening on SSL port while SSL handshaking
